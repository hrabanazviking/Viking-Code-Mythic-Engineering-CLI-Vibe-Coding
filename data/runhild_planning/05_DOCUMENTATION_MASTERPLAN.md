# Documentation Masterplan

## Core Docs to Create
- System architecture atlas.
- Data contracts and schema registry.
- Operational runbooks (incident, recovery, rollback).
- Testing strategy and quality gates.
- Lore governance policy.

## Core Docs to Improve
- Upgrade all module docs to include: purpose, inputs, outputs, failure modes.
- Add sequence diagrams for critical flows.
- Add troubleshooting decision trees.
- Add glossary for in-universe and technical vocabulary.

## Documentation Sprints
- Sprint Doc-1: Inventory all docs + identify stale pages.
- Sprint Doc-2: Build canonical architecture map.
- Sprint Doc-3: Fill runbooks and checklists.
- Sprint Doc-4: Add contributor onboarding and examples.

## Definition of Done for Docs
- Reviewed by engineering and narrative owners.
- Includes runnable examples where relevant.
- Linked from top-level index.
- Versioned and dated.
